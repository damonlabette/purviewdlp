How to create Endpoint DLP Demo environment and demonstrate print blocking policy

Authored by: Damon LaBette and Raquel Contreras

 

1)      Create demo tenant or use non-production tenant.

2)      Login to https://security.microsoft.com with admin credentials, go to advanced hunting, and wait for coffee cup to go away

3)      While waiting on coffee cup, install and setup new Windows 11 client with latest version of Office. Activate Office and Setup OneDrive as a user in tenant.

4)      Onboard the Windows 11 client into Defender XDR portal. (Settings -> Device management -> Onboarding)

5)      Turn on the following switches in Defender XDR portal (Settings -> General -> Advanced features): Unified audit log, Share endpoint alerts with Microsoft Compliance Center, and (optionally) Preview features.

6)      Login to https://compliance.microsoft.com and navigate to Settings > Device onboarding. Click the blue button to Turn on device onboarding

7)      While waiting for device monitoring to be turned on, start crafting DLP policy.

8)      In https://compliance.microsoft.com, navigate to Solutions > Data loss prevention > Policies. Create policy - > Custom - > Custom policy -> Next

9)      Name the new custom policy Endpoint DLP for SS and CC and give a good description.

10)  Click Next for Admin Units

11)  Uncheck everything except devices, and then click Next

12)  Define policy settings, click next

13)  Create rule

14)  Name the new rule policy Endpoint DLP rule for SS and CC and give a good description.

15)  Add condition -> Content contains

16)  Under Conditions -> Content Contains, select Sensitive info types Credit Card Number and U.S. Social Security Number

17)  Under Actions, Add an action,

 

18)  Set everything to Block with override

19)  Turn on user notifications

20)  Set incident alert severity level

21)  Click Save and it should look like this:

22)  Click Next, Turn the policy on immediately

 

Note: DLP policies should be tested in simulation mode first if planning to enable for all users. Another option is to turn on DLP policy with a small set of users that can be configured next to "Devices" to test DLP with those users before enabling to all. 

 

 

23)  Click Next

24)  Click Submit

25)  Select the vertical ellipses next to the new policy, and Move it to top (highest priority).

26)  It should now look like this:

27)  Navigate to Settings -> Device onboarding -> Devices.

 

 

 

 

 

 

28)  Wait for policy sync status on endpoint to complete or go into endpoint and force a policy sync.

a)      Policy not yet synced. b) Policy synced

29)  Prepare test document in Word with sample (not real) SS and CC data. Sample data can be obtained through 3rd party https://dlptest.com/sample-data/. Save document as hr_data.docx into user OneDrive.

30)  Start demonstrations of Block with Override. Print Word document to PDF.
